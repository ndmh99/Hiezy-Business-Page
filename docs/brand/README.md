# HIEZY Brand Design Guide

This folder is the source of truth for how HIEZY should look, sound, and behave across marketing pages, proposals, client-facing touchpoints, and future software products.

## What Is In Here

- `brand-foundation.md` defines the business positioning, audience, promise, and personality.
- `visual-identity.md` defines logo usage, color, typography, imagery, layout language, and accessibility standards.
- `voice-and-messaging.md` defines tone of voice, messaging pillars, copy rules, and CTA guidance.
- `ux-ui-guidelines.md` defines the user experience principles and interface standards for the HIEZY website and future digital products.

## Who This Is For

- Founders making brand and product decisions
- Designers building marketing pages or product UI
- Developers translating brand decisions into code
- Copywriters creating sales, landing-page, and support content

## How To Use The Guides

1. Start with `brand-foundation.md` before making any design or copy decisions.
2. Use `visual-identity.md` when building anything visual.
3. Use `voice-and-messaging.md` when writing headlines, CTAs, service descriptions, or emails.
4. Use `ux-ui-guidelines.md` when designing layouts, flows, components, and interactions.
5. Use [design-system/design-tokens.md](design-system/design-tokens.md) when converting the brand into implementation-ready tokens.

## Current Brand Signals Already Present In The Site

- Core promise: affordable, professional, easy-to-manage websites and software
- Trust markers: clear pricing, no hidden fees, full ownership, post-launch support
- Visual language: signature sky-blue brand accent (`#0EA5E9`), soft glass surfaces, high-contrast typography, premium editorial composition
- Primary type pairing: `Plus Jakarta Sans` for display/UI and `Source Serif 4` for warmth and editorial contrast

## Design Intent In One Sentence

HIEZY should feel like a sharp digital partner for ambitious small businesses: premium enough to build trust, simple enough to feel approachable, and clear enough to convert quickly.
