# HIEZY UX And UI Guidelines

## UX Vision

HIEZY interfaces should help people make confident decisions quickly. The experience should feel polished and intentional, but never confusing or overloaded.

## Core UX Principles

### 1. Reduce Decision Friction

Every page should make the next step obvious. Each section should answer one main question and point toward one clear action.

### 2. Build Trust Before Asking For Commitment

Use pricing clarity, process explanation, ownership language, portfolio proof, and support reassurance before high-friction asks.

### 3. Keep Interfaces Business-Readable

Clients should not need design literacy or technical literacy to understand what they are seeing.

### 4. Design For Momentum

Pages should move users steadily from orientation to trust to action.

## Layout Patterns

### Marketing Pages

- strong hero with one primary CTA and one secondary CTA
- immediate trust statement near the hero
- visible process explanation
- service or pricing blocks with concise comparison
- proof section before final contact conversion

### Product Or Dashboard UI

- prioritize clarity over novelty
- keep navigation obvious
- use cards and sectional grouping for scanability
- show status, progress, and next actions clearly

## Information Hierarchy

- Most important message first
- Supporting explanation second
- Detail only when needed
- One primary CTA per section
- One dominant focal point per viewport

## Interaction Rules

- Hover states should be subtle and fast.
- Focus states must be clearly visible.
- Animations should support orientation and delight lightly.
- Buttons should communicate hierarchy through color, weight, and contrast.
- Forms should be as short as possible while still collecting useful qualification details.

## Navigation Standards

- Keep top-level navigation concise.
- Use labels users already understand.
- Avoid hiding essential information behind interaction tricks.
- Mobile navigation should preserve access to pricing, services, and contact actions.

## CTA Hierarchy

### Primary CTA

Use filled accent buttons for the highest-value next action.

### Secondary CTA

Use outlined or soft-surface buttons for lower-friction exploration.

### Tertiary CTA

Use text links only when the action is clearly secondary.

## Card Design Standards

- Use rounded corners and soft borders.
- Maintain clear separation between title, price/value, and supportive detail.
- Avoid overfilling cards with too many bullet points.
- Use icons sparingly to anchor meaning.

## Forms

### Form Principles

- Ask only what helps respond well.
- Group fields logically.
- Use explicit labels, not placeholder-only inputs.
- Add reassuring microcopy where users may hesitate.

### Good Form Tone

- Tell users what happens after submission.
- Reinforce low pressure.
- Keep error and success states calm and specific.

## Responsive Behavior

- Mobile layouts should preserve hierarchy, not just stack everything blindly.
- Headlines may shorten on mobile if readability improves.
- Cards and buttons should remain easy to tap.
- Side-by-side comparisons should become clear stacked groups on smaller screens.

## Accessibility

- Ensure readable text sizing on mobile.
- Maintain strong contrast for core actions and body copy.
- Use semantic grouping and clear labels.
- Avoid relying on hover alone for important information.

## Content Density

HIEZY should generally prefer medium-low density. The interface should feel information-rich but not crowded.

## Design Patterns To Repeat

- hero split layouts
- clear service tier cards
- concise process timelines
- proof-driven testimonial or portfolio sections
- soft-glass containers with strong text hierarchy

## Design Patterns To Avoid

- giant paragraphs
- carousels that hide essential information
- more than two competing CTA styles in one section
- decorative motion that delays comprehension
- visually noisy dashboards

## Definition Of Done For HIEZY UI

The design is ready when:

- the offer is understood in under 5 seconds
- the next action is obvious
- the interface looks credible on desktop and mobile
- trust is reinforced without excessive copy
- visual polish supports conversion instead of distracting from it
